    生成器函数及其应用

    黎子 2023/11/12 JavaScript

    # 生成器函数 （Generator Function）

    # 定义和语法

    生成器函数（generator function）是一种特殊的函数，它可以在执行过程中暂停和恢复，返回一个可迭代的对象（iterable object）。生成器函数的语法是在函数关键字后面加一个星号（*），并使用 yield 关键字来返回中间值。

    function* gen() {
	console.log('start');
	yield 1;  // 暂停并返回 1
	console.log('resume');
	yield 2; // 暂停并返回 2
	console.log('end');
	return 3; // 结束并返回 3
}

    # yield*

    # 什么是 yield*

    yield* 是一个 Javascript 的语法特征，它可以用在生成器函数中，来委托给另一个可迭代对象或生成器函数。

    当我们在生成器函数中使用 yield* 关键字，我们可以将执行权交给另一个可迭代对象或生成器函数，直到它们迭代结束，然后再回到原来的生成器函数中。

    这样可以让我们更方便地组合多个可迭代对象或生成器函数，实现更复杂的逻辑和功能。

    # yield* 的语法和用法

    yield* 的语法如下：

    yield* iterable;

    其中，iterable 是一个可迭代对象或生成器函数。

    yield* 的用法有以下几种情况：

    • 当 iterable 是一个可迭代对象时，yield* 会遍历该对象，并依次返回它的每个元素。
    • 当 iterable 是一个生成器函数时，yield* 会调用该函数，并返回它的每个结果。
    • 当 iterable 是一个空对象时，yield* 不会产生任何结果。
    • 当 iterable 不是一个可迭代对象或生成器函数时，yield* 会抛出一个 TypeError 异常。

    # yield* 的示例

    # 使用 yield* 遍历数组

    可以使用 yield* 来遍历一个数组，并返回它的每个元素。 例如：

    // 定义一个生成器函数，使用 yield* 遍历数组
function* iterateArray(arr) {
	yield* arr;
}

// 调用 iterateArray([1, 2, 3])得到一个生成器对象
const iter = iterateArray([1, 2, 3]);

// 使用 for ... of 循环遍历生成器对象
for (let x of iter) {
  console.log(x);
}

// 输出结果为：
// 1
// 2
// 3
    # 使用 yield* 委托给另一个生成器函数

    可以使用 yield* 来委托给另一个生成器函数，并返回它的每个结果。 例如：

    // 定义一个生成器函数，返回 1 到 n 的自然数
function* naturalNumbers(n) {
	for (let i = 1; i <= n; ++i) {
		yield i;
	}
}

// 定义一个生成器函数，返回斐波那契数列的前 n 项
function* fibonacci(n) {
	let a = 0;
	let b = 1;
	for (let i = 0; i < n; ++i) {
		yield a;
		[a, b] = [b, a + b];
	}
}

// 定义一个生成器函数，使用 yield* 委托给其他两个生成器函数
function* combine(n) {
	// 先返回 1 到 n 的自然数
	yield* naturalNumbers(n);
	// 再返回斐波那契数列的前 n 项
	yield* fibonacci(n);
}


// 调用combine(5)得到一个生成器对象
const iter = combine(5);

// 使用for...of循环遍历生成器对象
for (let x of iter) {
  console.log(x);
}

// 输出结果为：
// 1
// 2
// 3
// 4
// 5
// 0
// 1
// 1
// 2
// 3
    # 使用 yield* 处理空对象

    使用 yield* 来处理空对象，并不产生任何结果。

    例如：

    // 定义一个生成器函数，使用 yield* 处理空对象
function* emptyObject() {
  yield* {};
}

// 调用 emptyObject() 得到一个生成器对象
const iter = emptyObject();

// 使用 for...of 循环遍历生成器对象
for (let x of iter) {
  console.log(x);
}

// 没有任何输出结果
    # 使用 yield* 处理非可迭代对象

    使用 yield* 来处理非可迭代对象，会抛出一个 TypeError 异常。

    例如：

    // 定义一个生成器函数，使用 yield* 处理非可迭代对象
function* nonIterable() {
  yield* 1;
}

// 调用 nonIterable() 得到一个生成器对象
const iter = nonIterable();

// 使用 for...of 循环遍历生成器对象
for (let x of iter) {
  console.log(x);
}

// 抛出异常：TypeError: number 1 is not iterable (cannot read property Symbol(Symbol.iterator))

    # 创建和遍历生成器对象

    生成器函数可以通过调用它来创建一个生成器对象（generator object），它遵循迭代器协议（iterator protocol），也就是说，它有一个 next() 方法，可以返回一个对象，包含 value 和 done 两个属性，分别表示当前的值和是否结束。

    function* gen() {
	console.log('start');
	yield 1;  // 暂停并返回 1
	console.log('resume');
	yield 2; // 暂停并返回 2
	console.log('end');
	return 3; // 结束并返回 3
}

let g = gen(); // 创建一个生成器对象
console.log(g.next()); // { value: 1, done: false }
console.log(g.next()); // { value: 2, done: false }
console.log(g.next()); // { value: 3, done: true }
console.log(g.next()); // { value: undefined, done: true }

    生成器对象也遵循可迭代协议（iterable protocol），也就是说，它有一个 Symbol.iterator 属性，指向自身。因此，可以使用 for ... of 循环或扩展运算符（...）来遍历生成器对象。

    for (let x of gen()) {
  console.log(x); // 依次打印 1, 2, 3
}

let arr = [...gen()]; // 将生成器对象转为数组
console.log(arr); // [1, 2, 3]

    # 实现惰性求值

    生成器函数可以用来实现惰性求值（lazy evaluation），也就是说，只在需要的时候计算值，而不是一次性计算所有的值。这样可以节省内存和性能，也可以处理无限序列。

    例如，下面代码定义了一个斐波那契数列的生成器函数：

    function* fibonacci() {
	let a = 0;
	let b = 1;
	while (true) {
		let c = a + b;
		yield c;
		a = b;
		b = c;
	}
}

    这个函数可以无限地产生斐波那契数列的项，但是只有在调用 next() 方法或遍历时才会计算下一个值。因此，我们可以按需获取任意个斐波那契数的值，而不会造成内存溢出。

    let f = fibonacci();
console.log(f.next().value); // 1
console.log(f.next().value); // 2
console.log(f.next().value); // 3
console.log(f.next().value); // 5
console.log(f.next().value); // 8

// 获取前十个斐波那契数
let arr = [];
for (let i = 0; i < 10; i++) {
  arr.push(f.next().value);
}
console.log(arr); // [13, 21, 34, 55, 89, 144, 233, 377, 610, 987]

    # 实现异步编程

    生成器函数还可以用来实现异步编程，也就是说可以在等待异步操作的结果时暂停执行，并在异步操作完成后恢复执行。这样可以避免回调地狱或复杂的 Promise 链，使得异步代码看起来像同步代码一样简洁清晰。

    例如：下面的代码使用了一个名为 co 的库，它可以自动执行生成器函数，并将 yield 后面的异步操作包装成 Promise 对象，等待其状态变为 resolved 后再继续执行。

    const co = require('co');
const fetch = require('node-fetch');

function* fetchUser() {
	let response = yield fetch('https://api.github.com/users/github');
	let user = yield response.json();
	console.log(user.name);
}

co(fetchUser);

    # 生成器函数面试题

    写一个生成器函数，实现 range 功能，可以指定开始，结束和步长，返回一个序列。

    function* range(start, end, step) {
	while (start < end) {
		yield start;
		start += step;
	}
}

// @test
let r = range(1, 10, 2);
console.log([...r]); // [1, 3, 5, 7, 9]

    写一个生成器函数，实现 flatten 功能，可以将一个嵌套的数组扁平化。

    function* flatten(arr) {
	for (let item of arr) {
		if (Array.isArray(item)) {
			yield* flatten(item); // 使用 yield* 委托给另一个生成器
		} else {
			yield item;
		}
	}
}

// @test
let arr = [1, [2, [3, 4], 5], 6];;
console.log([...flatten(arr)]); // [1, 2, 3, 4, 5, 6]
