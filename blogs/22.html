    Javascript Proxy

    黎子 2023/11/13 Javascript

    # JavaScript Proxy

    在 JavaScript 中，代理（Proxy）是一种对象，它允许我们拦截并定义 JavaScript 操作的行为。代理对象包装了另一个对象（目标对象），并可以拦截对目标对象的操作，提供自定义行为。代理是 ES6 中的一个新增特性，并且是 ECMAScript 6 中新增的一个标准内置对象。

    # 创建Proxy对象

    通过 new Proxy() 构造函数创建 Proxy 对象，该函数接收两个参数：目标对象 target 和一个处理器对象 handler。处理器对象 handler 包含一组函数，这些函数都是用于拦截对目标对象的各种操作，并在拦截到这些操作时执行自定义行为。Proxy 对象会在访问或修改目标对象时触发处理器中对应的拦截函数，从而实现对目标对象操作的自定义控制。

    下面是创建代理对象的实例代码：

    const proxy = new Proxy(target, handler);
    • target：目标对象，可以是任何类型的对象，包括函数、数组和普通对象。
    • handler：处理程序对象，它定义了用于拦截操作的方法。
    let obj = { a: 1 };
let proxy = new Proxy(obj, {
	get(target, propKey) {
		console.log(`正在访问属性${propKey}`);
		return target[propKey];
	},
	set(target, propKey, value) {
		console.log(`正在设置属性${propKey}的值为${value}`);
		target[propKey] = value;
		return true;
	}
});

proxy.a;  // 输出：正在访问属性a
proxy.b = 2;  // 输出：正在设置属性b的值为2

    在上面的代码中，我们创建了一个普通对象 obj，将它包装在一个代理对象中，并定义了一组拦截器函数。在 get 函数中，我们记录了属性访问的信息，并返回属性的值。在 set 函数中，我们记录了属性设置的信息，并将属性值设置到目标对象中。通过这些拦截器函数，我们实现了对目标对象属性访问和设置的自定义控制。

    # 拦截器函数

    通过 Proxy 对象可以拦截对目标对象的属性访问、设置、删除等操作，并在拦截到这些操作时执行自定义行为。处理器对象 handler 包含一组函数，这些函数都是用于拦截对目标对象的各种操作，并在拦截到这些操作时执行自定义行为。下面是常见的拦截器函数：

    # get(target, propKey, receiver)

    get 拦截器函数用于拦截对目标对象属性的读取操作。它接收三个参数：

    • target：目标对象，即被代理的对象
    • propKey：属性名称，即要读取的属性的名称
    • receiver：Proxy 或继承 Proxy 的对象，通常是 proxy

    下面是 get 拦截器函数的实例代码：

    let obj = { a: 1 };
let proxy = new Proxy(obj, {
	get(target, propKey) {
		console.log(`正在访问属性${propKey}`);
		return target[propKey];
	}
});

proxy.a;  // 输出：正在访问属性a

    在上面的代码中，我们使用了 get 拦截器函数，对目标对象的属性读取进行了拦截。在拦截到属性读取时，我们记录了属性读取的信息，并返回属性的值。

    # set(target, propKey, value, receiver)

    set 拦截器函数用于；拦截对目标对象属性的设置操作。它接收四个参数：

    • target：目标对象，即被代理的对象
    • propKey：属性名称，即要设置的属性的名称
    • value：属性值，即要设置的属性的值
    • receiver：Proxy 或继承代理 Proxy 的对象，

    下面是 set 拦截器函数的示例代码：

    let obj = { a: 1 };
let proxy = new Proxy(obj, {
set(target, propKey, value, receiver) {
    console.log(`正在设置属性${propKey}的值为${value}`);
    target[propKey] = value;
    return true;
  }
});

proxy.b = 2; // 输出：正在设置属性b的值为2
console.log(proxy.b); // 输出：2

    在上面的代码中，我们使用了 set 拦截器函数，对目标对象的属性设置进行了拦截。在拦截到属性设置时，我们记录了属性设置的信息，并将属性值设置到目标对象中。 需要注意的是，set 拦截器函数必须返回一个布尔值，表示属性是否设置成功。如果返回 false，则表示属性设置失败，否则表示属性设置成功

    # has(target, propKey)

    has 拦截器函数用于拦截对目标对象属性是否存在的判断操作。它接收两个参数：

    • target：目标对象，即被代理的对象。
    • propKey：属性名称，即要判断的属性的名称。

    下面是 has 拦截器函数的示例代码：

    let obj = { a: 1 };
let proxy = new Proxy(obj, {
  has(target, propKey) {
    console.log(`正在判断属性${propKey}是否存在`);
    return propKey in target;
  }
});

console.log('a' in proxy); // 输出：正在判断属性a是否存在，true

    在上面的代码中，我们使用了 has 拦截器函数，对目标对象的属性是否存在进行了拦截。在拦截到属性是否存在的判断操作时，我们记录了属性是否存在的信息，并返回属性是否存在的结果。

    # deleteProperty(target, propKey)

    deleteProperty 拦截器函数用于拦截对目标对象属性的删除操作。它接收两个参数：

    • target：目标对象，即被代理的对象。
    • propKey：属性名称，即要删除的属性的名称。

    下面是 deleteProperty 拦截器函数的示例代码：

    let obj = { a: 1 };
let proxy = new Proxy(obj, {
  deleteProperty(target, propKey) {
    console.log(`正在删除属性${propKey}`);
    delete target[propKey];
    return true;
  }
});

delete proxy.a; // 输出：正在删除属性a
console.log(proxy.a); // 输出：undefined

    在上面的代码中，我们使用了 deleteProperty 拦截器函数，对目标对象的属性删除进行了拦截。在拦截到属性删除时，我们记录了属性删除的信息，并将属性从目标对象中删除。

    # apply(target, thisArg, argList)

    apply 拦截器函数用于拦截对目标对象的函数调用操作。它接收三个参数：

    • target：目标对象，即被代理的函数。
    • thisArg：函数调用时的 this 对象。
    • argList：函数调用时传入的参数列表。

    下面是 apply 拦截器函数的示例代码：

    function sum(a, b) {
  return a + b;
}

let proxy = new Proxy(sum, {
  apply(target, thisArg, argList) {
    console.log(`正在调用函数${target.name}`);
    return target.apply(thisArg, argList);
  }
});

proxy(1, 2); // 输出：正在调用函数sum，3

    在上面的代码中，我们使用了 apply 拦截器函数，对目标函数的调用进行了拦截。在拦截到函数调用时，我们记录了函数调用的信息，并将调用传递给目标函数。

    # construct(target, argList)

    construct 拦截器函数用于拦截对目标对象的构造函数调用操作。它接收两个参数：

    • target：目标对象，即被代理的构造函数。
    • argList：构造函数调用时传入的参数列表。

    下面是 construct 拦截器函数的示例代码：

    class Person {
  constructor(name) {  
		this.name = name;  
	}  
}

let proxy = new Proxy(Person, {  
	construct(target, argList) {  
		console.log(`正在使用构造函数${target.name}`);  
		return new target(...argList);  
	}  
});

let p = new proxy('Tom'); // 输出：正在使用构造函数Person  
console.log(p.name); // 输出：Tom

    在上面的代码中，我们使用了 construct 拦截器函数，对目标构造函数的调用进行了拦截。在拦截到构造函数调用时，我们记录了构造函数调用的信息，并返回一个新的对象。

    # 拦截器函数的返回值

    拦截器函数的返回值决定了对目标对象操作的成功或失败。下面是拦截器函数的返回值的含义：

    • get 拦截器函数：返回属性值
    • set 拦截器函数：返回布尔值，表示属性是否设置成功
    • has 拦截器函数：返回布尔值，表示属性是否存在
    • deleteProperty 拦截器函数：返回布尔值，表示属性是否删除成功
    • apply 拦截器函数：返回函数调用的结果
    • construct 拦截器函数：返回一个对象，通常是目标对象的一个实例对象

    # 常见用法

    # 属性拦截

    通过代理对象可以拦截对目标对象的属性访问、设置、删除等操作，并在拦截到这些操作时执行自定义行为。例如，可以通过代理对象实现禁止修改对象的某些属性、对对象进行类型检查、属性访问记录等功能。

    let obj = { a: 1 };
let proxy = new Proxy(obj, {
  get(target, propKey) {
    console.log(`正在访问属性${propKey}`);
    return target[propKey];
  },
  set(target, propKey, value) {
    if (propKey === 'a') {
      throw new Error('属性a不能修改');
    }
    if (typeof value !== 'number') {
      throw new TypeError('属性值必须是数字');
    }
    target[propKey] = value;
  },
  deleteProperty(target, propKey) {
    console.log(`正在删除属性${propKey}`);
    delete target[propKey];
  }
});

console.log(proxy.a); // 输出：正在访问属性a，1
proxy.b = 2; // 输出：b=2
proxy.c = 'hello'; // 抛出TypeError异常
delete proxy.a; // 输出：正在删除属性a

    在上面的代码中，我们创建了一个普通对象obj，将它包装在一个代理对象中，并定义了一组拦截器函数。在get函数中，我们记录了属性访问的信息，并返回属性的值。在set函数中，我们对属性设置进行了类型检查，如果类型不符合要求就抛出异常，否则将属性值设置到目标对象中。在deleteProperty函数中，我们记录了属性删除的信息，并将属性从目标对象中删除。通过这些拦截器函数，我们实现了对目标对象属性访问、设置和删除的自定义控制。

    # 数据绑定

    使用代理对象可以实现数据绑定，即当数据发生变化时，自动更新视图。例如，可以使用代理对象实现对数组和对象的监听，当数组或对象发生变化时，自动更新视图。

    let data = { name: '张三', age: 18 };
let proxy = new Proxy(data, {
	set(target, propKey, value) {
		target[propKey] = value;
		console.log(`属性${propKey}的值被修改为${value}`);
		render();
		return true;
	}
});

function render() {
	console.log(`当前数据：${JSON.stringify(proxy)}`);
}

render(); // 输出：当前数据：{"name":"张三","age":18}
proxy.name = '李四'; // 输出：属性name的值被修改为李四，当前数据：{"name":"李四","age":18}

    在上面的代码中，我们创建了一个普通对象data，将它包装在一个代理对象中，并定义了一个set函数，用于在属性修改时自动更新视图。在set函数中，我们记录了属性修改的信息，并调用render函数更新视图。通过这个代理对象，我们实现了对数据的监听和自动更新视图的功能。

    # 安全检查

    使用代理对象可以进行安全检查，例如，可以通过代理对象实现不信任代码的隔离、防止跨站脚本攻击等功能。

    let obj = { a: 1 };
let proxy = new Proxy(obj, {
  set(target, propKey, value) {
    if (propKey === 'a') {
      throw new Error('属性a不能修改');
    }
    if (typeof value !== 'number') {
      throw new TypeError('属性值必须是数字');
    }
    target[propKey] = value;
  }
});

try {
  proxy.a = 2;
} catch (e) {
  console.error(e.message); // 输出"属性a不能修改"
}

try {
  proxy.b = 'hello';
} catch (e) {
  console.error(e.message); // 输出"属性值必须是数字"
}

console.log(proxy.a); // 输出 1
console.log(proxy.b); // 输出 "undefined"

    在上面的代码中，我们创建了一个普通对象obj，将它包装在一个代理对象中，并定义了一个set函数，用于对属性设置进行安全检查。在set函数中，我们禁止修改属性a，并对属性值进行类型检查。在try...catch语句中，我们通过代理对象修改了属性a的值，这里会抛出一个异常。通过这个代理对象，我们实现了对不信任代码的隔离和安全检查的功能。最后，我们输出了代理对象的属性值，验证了代理对象的确起到了过滤作用。

    # 总结

    代理（Proxy）是 JavaScript 中的一个内置对象，它允许我们拦截并自定义 JavaScript 操作的行为。通过代理对象，我们可以拦截对目标对象的属性访问、设置、删除等操作，并在拦截到这些操作时执行自定义行为。代理对象是通过 new Proxy() 构造函数来创建的，该函数接收两个参数：目标对象 target 和一个处理器对象 handler。处理器对象 handler 包括一组函数，用于拦截对目标对象的各种操作，并在拦截到这些操作时执行自定义行为。常见的拦截器函数包括 get、set、has、deleteProperty、apply 和 construct。拦截器函数的返回值决定了对目标对象操作的成功或失败。
